# 📘 DBMS Architecture Components

*(FAANG + Interview + GitHub Ready Notes)*

---

# 🔥 Introduction

DBMS Architecture explains how a database system internally processes, manages, stores, and retrieves data.

It shows:
- how queries are executed
- how data is stored
- how memory is managed
- how security and transactions are controlled

This is one of the most important DBMS interview topics.

---

# 📌 Major Components of DBMS Architecture

1️⃣ Query Processor  
2️⃣ Storage Manager  
3️⃣ Disk Storage  

---

# 🧠 Overall Flow

```text
User Query
     ↓
Query Processor
     ↓
Storage Manager
     ↓
Disk Storage
     ↓
Result Returned to User
```

---

# 1️⃣ Query Processor

## Definition

The Query Processor interprets and executes SQL queries.

It converts high-level SQL queries into low-level instructions understandable by the DBMS.

---

# 📌 Responsibilities

✅ Query interpretation  
✅ Query optimization  
✅ Query execution  
✅ Choosing efficient execution plans  

---

# 📌 Components of Query Processor

1. DML Compiler  
2. Query Evaluation Engine  

---

# 🔥 1. DML Compiler

## Definition

The DML Compiler converts SQL queries into low-level execution instructions.

---

# 📌 Example Query

```sql
SELECT * FROM Students
WHERE Marks > 80;
```

The DML Compiler:
- analyzes query syntax
- checks validity
- converts query into execution plan

---

# 📌 Responsibilities

✅ Syntax checking  
✅ Query translation  
✅ Query optimization preparation  

---

# 📌 Important Point

DML stands for:

```text
Data Manipulation Language
```

Examples:
- SELECT
- INSERT
- UPDATE
- DELETE

---

# 🎯 Interview Question

## Q. What is the role of DML Compiler?

### Answer

The DML Compiler converts SQL queries into low-level instructions that can be executed by the database system.

---

# 🔥 2. Query Evaluation Engine

## Definition

The Query Evaluation Engine actually executes the query execution plan generated by the compiler.

---

# 📌 Responsibilities

✅ Execute SQL queries  
✅ Retrieve required records  
✅ Perform joins and filtering  
✅ Access storage manager  

---

# 📌 Example

For query:

```sql
SELECT * FROM Students
WHERE Marks > 80;
```

The engine:
- searches records
- applies conditions
- returns results

---

# 📌 Optimization Example

Instead of scanning the entire table, it may:
- use indexes
- use hashing
- choose fastest path

---

# 🎯 Interview Question

## Q. Difference between DML Compiler and Query Evaluation Engine?

### Answer

| DML Compiler | Query Evaluation Engine |
|---|---|
| Converts query into execution plan | Executes execution plan |
| Performs translation | Performs actual execution |

---

# 2️⃣ Storage Manager

## Definition

The Storage Manager manages how data is stored and retrieved from disk.

It acts as a bridge between:
- query processor
- physical storage

---

# 📌 Responsibilities

✅ Disk interaction  
✅ Memory management  
✅ Authorization  
✅ Transaction handling  
✅ File organization  

---

# 📌 Components of Storage Manager

1. Buffer Manager  
2. File Manager  
3. Authorization & Integrity Manager  
4. Transaction Manager  

---

# 🔥 1. Buffer Manager

## Definition

The Buffer Manager manages temporary memory (RAM) used to store frequently accessed data.

---

# 📌 Why Needed?

Reading from disk repeatedly is slow.

So DBMS:
- loads frequently used data into memory
- reduces disk access

---

# 📌 Responsibilities

✅ Memory allocation  
✅ Cache management  
✅ Faster data access  

---

# 📌 Real Life Example

Like keeping frequently used books on a study table instead of searching the library every time.

---

# 🎯 Interview Question

## Q. Why is Buffer Manager important?

### Answer

Because accessing RAM is much faster than accessing disk storage, improving DBMS performance.

---

# 🔥 2. File Manager

## Definition

The File Manager manages physical database files stored on disk.

---

# 📌 Responsibilities

✅ File organization  
✅ Disk space allocation  
✅ Record management  
✅ Storage structure management  

---

# 📌 Example

Managing:
- student.db
- employee.dat
- index files

---

# 🎯 Interview Question

## Q. What is the role of File Manager?

### Answer

The File Manager manages storage allocation, file organization, and physical record storage in the database.

---

# 🔥 3. Authorization & Integrity Manager

## Definition

This component controls:
- security
- permissions
- integrity constraints

---

# 📌 Responsibilities

✅ User authentication  
✅ Access control  
✅ Constraint checking  
✅ Data validation  

---

# 📌 Examples of Constraints

- PRIMARY KEY
- FOREIGN KEY
- UNIQUE
- NOT NULL

---

# 📌 Example Permission

```sql
GRANT SELECT ON Students TO user1;
```

---

# 🎯 Interview Question

## Q. Which DBMS component handles security?

### Answer

Authorization & Integrity Manager.

---

# 🔥 4. Transaction Manager

## Definition

The Transaction Manager ensures safe and consistent execution of transactions.

---

# 📌 What is a Transaction?

A transaction is a logical unit of work.

Example:
- money transfer
- online payment
- ticket booking

---

# 📌 Example

Bank transfer:

```text
A → B transfer ₹1000
```

Steps:
1. Deduct from A
2. Add to B

If one step fails:
- rollback occurs

---

# 📌 Responsibilities

✅ Transaction control  
✅ Concurrency control  
✅ Rollback management  
✅ Recovery management  
✅ ACID property support  

---

# 🎯 Interview Question

## Q. Why is Transaction Manager important?

### Answer

It ensures data consistency and reliability during concurrent or failed transactions.

---

# 3️⃣ Disk Storage

## Definition

Disk Storage contains all permanent database data stored physically on disk.

---

# 📌 Components of Disk Storage

1. Actual Data  
2. Indices  
3. Data Dictionary  
4. Statistical Data  

---

# 🔥 1. Actual Data

## Definition

The real records stored inside database tables.

---

# 📌 Example

| ID | Name | Marks |
|---|---|---|
| 101 | Riya | 85 |

---

# 🎯 Interview Question

## Q. What is actual data in DBMS?

### Answer

Actual data refers to the real records stored inside database tables.

---

# 🔥 2. Indices

## Definition

Indexes improve searching speed in databases.

---

# 📌 Example

Like a book index:
- directly jumps to required topic

instead of reading the whole book.

---

# 📌 Advantages

✅ Faster searching  
✅ Faster retrieval  
✅ Better performance  

---

# 🎯 Interview Question

## Q. Why are indexes important?

### Answer

Indexes reduce search time and improve query performance.

---

# 🔥 3. Data Dictionary

## Definition

The Data Dictionary stores metadata.

Metadata means:
> “Data about data”

---

# 📌 Stores Information About

✅ Table names  
✅ Column names  
✅ Data types  
✅ Constraints  
✅ Relationships  

---

# 📌 Example

```text
Student Table
- ID → INT
- Name → VARCHAR
```

---

# 🎯 Interview Question

## Q. What is metadata?

### Answer

Metadata is information describing database structure and properties.

---

# 🔥 4. Statistical Data

## Definition

Statistical Data stores information used for query optimization.

---

# 📌 Examples

- number of rows
- table size
- data distribution
- index statistics

---

# 📌 Why Important?

Helps DBMS choose:
- fastest query execution plan
- best indexes
- optimized joins

---

# 🎯 Interview Question

## Q. Why does DBMS store statistical data?

### Answer

To optimize query execution and improve performance.

---

# 🧠 Full Query Execution Flow

```text
1. User sends SQL query
2. DML Compiler translates query
3. Query Evaluation Engine executes query
4. Storage Manager accesses data
5. Buffer Manager checks memory
6. File Manager accesses disk if needed
7. Result returned to user
```

# 🚀 FAANG-Level Conceptual Questions

---

## Q1. Why is query optimization important?

### Answer

Efficient query optimization:
- reduces execution time
- minimizes disk access
- improves scalability

---

## Q2. Why is disk access expensive?

### Answer

Disk access is much slower than RAM access.

Therefore DBMS uses:
- buffering
- caching
- indexing

to improve performance.

---

## Q3. Why are transactions critical in banking systems?

### Answer

Transactions ensure:
- consistency
- reliability
- rollback support
- concurrency control

during financial operations.
